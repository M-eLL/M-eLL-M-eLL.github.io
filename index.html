Recent Projects

In.security

Encryption-protected journaling application

Unload without fear of judgement! Through use of client-side ciphering, entries cannot be decrypted without the unique passphrase assigned to it upon creation. A custom algorithm (based on character count and the average person's reading pace) will ensure the entry only decrypts long enough to read once so that the user can step away from the app with a peace of mind. After a number of incorrect password attempts, "panic mode" is set off and the entry will be locked in a vault and then deleted if the user's session password is not entered.

Javascript | Express | PostgreSQL | React | Redux | Sequelize | CSS3 | HTML5 | Git | NPM | CryptoJS | Heroku
